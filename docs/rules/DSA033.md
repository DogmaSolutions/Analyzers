# DSA033

File exceeds maximum line count.

| Property     | Value                                       |
|:-------------|:--------------------------------------------|
| Rule ID      | DSA033                                       |
| Title        | File exceeds maximum line count              |
| Category     | Code Smells                                  |
| Severity     | ⚠ Warning                                    |
| Refactoring available | ✅                                  |
| Code-review tools available | ✅                            |
| Is enabled   | ✅                                            |

## 💡 Rationale

This rule fires when a source file exceeds a configurable maximum number of lines. Large files are harder to navigate, review, and maintain. They often indicate that the file contains too many responsibilities or too many types.

The maximum line count is configurable via `.editorconfig`. The default threshold is **500** lines.


## 🔐 Security implications

Excessively large files increase the risk that security-sensitive code goes unnoticed during code review. When a file contains thousands of lines, reviewers may skim or skip sections, potentially missing vulnerabilities, improper access controls, or unsafe data handling. Keeping files small and focused improves the likelihood that all code paths receive adequate review attention.

## 📖 Knowledge base

- [MITRE, CWE-1080: Source Code File with Excessive Number of Lines of Code](https://cwe.mitre.org/data/definitions/1080.html) — files with too many lines are harder to understand and maintain
- [Single Responsibility Principle](https://en.wikipedia.org/wiki/Single-responsibility_principle) — each file should have one reason to change
- [DSA032: Avoid duplicated string literals in the same method body](DSA032.md) — related code smell rule


## 🔦 Code sample

```csharp
// File: Models.cs (exceeds max_lines threshold)
namespace MyApp
{
    public class User
    {
        public int Id { get; set; }
        public string Name { get; set; }
        // ... many properties
    }

    public class Order
    {
        public int Id { get; set; }
        // ... many properties
    }

    public class Product
    {
        public int Id { get; set; }
        // ... many properties
    }

    // ... hundreds more lines with additional types
}
```

```cs

// Default threshold: 500 lines
// The diagnostic is reported on the first line of the file
```

## ✅ Not matched patterns (code is OK)

```cs
// Files at or below the threshold do not trigger the diagnostic

```

## 🛠️ Included Code Fixer and Mitigation

The code fixer offers one action when the file contains **two or more top-level types** (classes, structs, interfaces, enums, records):

**Split file into N files (one per type)**

Each top-level type is extracted into its own file named `TypeName.cs`. The original file is renamed to match its remaining type. All `using` directives are conservatively copied to each new file. Nested types stay inside their parent.

```cs
// Before: Models.cs (3 types in one file)
using System;

namespace MyApp
{
    public class User { ... }
    public class Order { ... }
    public class Product { ... }
}

// After: User.cs
using System;

namespace MyApp
{
    public class User { ... }
}

// After: Order.cs
using System;

namespace MyApp
{
    public class Order { ... }
}

// After: Product.cs
using System;

namespace MyApp
{
    public class Product { ... }
}
```

If the file contains only **one** top-level type, no code fix is offered (splitting wouldn't reduce the file).

## 🔧 Included Refactoring (DSR003)

The same "Split file into N files" action is also available as a **refactoring** from the lightbulb menu, without requiring the DSA033 diagnostic to fire. This is useful when you want to split a file that is below the threshold or when the diagnostic severity is set to `none`.

## 📝 Included tools for code review

The `[DSA033] Add comment for code review` action inserts a structured block comment above the first line. The comment includes the rule ID, enriched category, title with documentation link, and relevant external references.

## ⚙️ Rule configuration

### Severity
In order to change the severity level of this rule, change/add this line in the `.editorconfig` file:

```ini
# DSA033: File exceeds maximum line count
dotnet_diagnostic.DSA033.severity = error
```

### Maximum line count
The threshold is configurable via `.editorconfig`:

```ini
# Default is 500. Set higher for more lenient checks, lower for stricter.
dotnet_diagnostic.DSA033.max_lines = 500
```
