# DSA032

Avoid duplicated string literals in the same method body.

| Property     | Value                                                                                          |
|:-------------|:-----------------------------------------------------------------------------------------------|
| Rule ID      | DSA032                                                                                        |
| Title        | Avoid duplicated string literals in the same method body                                      |
| Category     | Code Smells                                                                                   |
| Severity     | Warning                                                                                       |
| Code fix     | Yes                                                                                           |
| Is enabled   | Yes                                                                                           |

## Description

This rule fires when the same string literal appears more than `max_duplications` times in the same method body (including constructors). Repeated string literals reduce maintainability: if the value needs to change, every occurrence must be found and updated, risking inconsistencies and subtle bugs.

Only string literals whose length meets or exceeds `min_string_length` are checked.

## Configuration

Both thresholds are configurable via `.editorconfig`:

```
# Maximum number of times a string literal may appear before the rule fires. Default is 2.
dotnet_diagnostic.DSA032.max_duplications = 2

# Minimum string length to consider for duplication analysis. Default is 5.
dotnet_diagnostic.DSA032.min_string_length = 5
```

### Ignoring specific strings

Specific string values can be excluded from analysis by listing them in a `DSA032_IgnoredStrings.txt` file, one string per line. Lines starting with `#` are treated as comments and ignored. Empty lines are ignored. The file is optional.

Add the file to your project as an `AdditionalFile`:

```xml
<ItemGroup>
  <AdditionalFiles Include="DSA032_IgnoredStrings.txt" />
</ItemGroup>
```

To share a single file across all projects in a solution:

```xml
<ItemGroup>
  <AdditionalFiles Include="$(SolutionDir)DSA032_IgnoredStrings.txt" />
</ItemGroup>
```

Example `DSA032_IgnoredStrings.txt`:

```
# Configuration keys that are intentionally repeated
ConnectionStrings:Default
AppSettings:Environment

# Test fixture values
TestCategory:Integration
```

### Ignoring files by name pattern

Entire source files can be excluded from analysis by file name pattern. The analyzer has built-in defaults that skip well-known designer-generated files:

- `*.Designer.cs`
- `*.generated.cs`
- `*.g.cs`
- `*.g.i.cs`

To exclude additional file name patterns, create a `DSA032_IgnoredFileNames.txt` file with one glob pattern per line. Each pattern supports multiple `*` wildcards and is matched case-insensitively against the file name (not the full path). Lines starting with `#` are comments. The file is optional; when present, its patterns are **added** to the built-in defaults.

```xml
<ItemGroup>
  <AdditionalFiles Include="DSA032_IgnoredFileNames.txt" />
</ItemGroup>
```

Example `DSA032_IgnoredFileNames.txt`:

```
# Auto-generated API client code
*ApiClient*.cs

# T4 template output
*.tt.cs
```

### Ignored base types

Classes inheriting (directly or indirectly) from certain framework base types are automatically excluded, regardless of file name. These classes contain repeated string literals by design and should not be flagged:

- `Microsoft.EntityFrameworkCore.Migrations.Migration` — EF Core migration classes (`Up`/`Down` methods repeat table and column names)
- `Microsoft.EntityFrameworkCore.Infrastructure.ModelSnapshot` — EF Core model snapshots (auto-generated, repeat all entity and property names)
- `System.Data.Entity.Migrations.DbMigration` — EF6 migration classes (`Up`/`Down` methods repeat table and column names)

## Matched patterns

```cs
public void Process()
{
    var a = "ConnectionStrings:Secret";     // DSA032
    var b = "ConnectionStrings:Secret";     // DSA032
    var c = "ConnectionStrings:Secret";     // DSA032
}
```

## Not matched patterns

```cs
// String appears only twice (within default max_duplications = 2)
public void Process()
{
    var a = "ConnectionStrings:Secret";     // OK
    var b = "ConnectionStrings:Secret";     // OK
}

// String is shorter than min_string_length (default 5)
public void Process()
{
    var a = "key";    // OK
    var b = "key";    // OK
    var c = "key";    // OK
}

// Strings are in different methods
public void Method1()
{
    var a = "ConnectionStrings:Secret";
}
public void Method2()
{
    var b = "ConnectionStrings:Secret";
}
```

## Fix / Mitigation

The code fixer offers two actions:

**1. Extract into local constant** (method body)

Extracts the duplicated string into a `const string` local variable within the method body:

```cs
// Before:
public void Process()
{
    var a = "ConnectionStrings:Secret";
    var b = "ConnectionStrings:Secret";
    var c = "ConnectionStrings:Secret";
}

// After:
public void Process()
{
    const string ConnectionStringsSecret = "ConnectionStrings:Secret";
    var a = ConnectionStringsSecret;
    var b = ConnectionStringsSecret;
    var c = ConnectionStringsSecret;
}
```

**2. Extract into class field constant**

Extracts the duplicated string into a `private const string` field on the containing class:

```cs
// Before:
public class MyService
{
    public void Process()
    {
        var a = "ConnectionStrings:Secret";
        var b = "ConnectionStrings:Secret";
        var c = "ConnectionStrings:Secret";
    }
}

// After:
public class MyService
{
    private const string ConnectionStringsSecret = "ConnectionStrings:Secret";
    public void Process()
    {
        var a = ConnectionStringsSecret;
        var b = ConnectionStringsSecret;
        var c = ConnectionStringsSecret;
    }
}
```

The generated constant name uses PascalCase, formed by concatenating the words and tokens from the string value. Non-alphanumeric characters are treated as word separators.

## Rule configuration

In order to change the severity level of this rule, change/add this line in the `.editorconfig` file:

```
# DSA032: Avoid duplicated string literals in the same method body
dotnet_diagnostic.DSA032.severity = warning
```
