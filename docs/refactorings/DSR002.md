# DSR002

Extract any string literal to a local constant or class field constant.

| Property     | Value                                                                                          |
|:-------------|:-----------------------------------------------------------------------------------------------|
| Refactoring ID | DSR002                                                                                      |
| Title        | Extract string literal to constant                                                             |
| Language     | C#                                                                                             |

## Description

This refactoring extracts a string literal to a named constant, improving readability and reducing the risk of typos when the same string appears in multiple places. All matching occurrences within the current scope are replaced.

Unlike the [DSA032](../rules/DSA032.md) analyzer (which only fires when a string exceeds a configured duplication threshold), this refactoring is available on **any** non-empty string literal — including single occurrences. It appears in the IDE lightbulb/screwdriver menu whenever the cursor is on a string literal.

## Trigger

The refactoring is offered when the cursor is on a string literal (`"..."`) that meets **all** of the following conditions:

1. The literal is a regular string literal (not an interpolated string, not a verbatim interpolated string)
2. The string value is non-empty
3. The literal is inside a class, struct, or record type declaration

## Actions offered

### Extract to local constant

Available when the literal is inside a method or constructor with a block body. Creates a `const string` local variable before the first statement that uses the string, and replaces all matching string literals within the method body.

```csharp
// Before:
public void Process()
{
    var a = "ConnectionStrings:Default";
    var b = "ConnectionStrings:Default";
}

// After:
public void Process()
{
    const string ConnectionStringsDefault = "ConnectionStrings:Default";
    var a = ConnectionStringsDefault;
    var b = ConnectionStringsDefault;
}
```

### Extract to class field constant

Always available when the literal is inside a type. Creates a `private const string` field before the first method or constructor, and replaces all matching string literals within the current method body (or the single targeted literal for expression-bodied members).

```csharp
// Before:
public class MyService
{
    public void Process()
    {
        var a = "ConnectionStrings:Default";
    }
}

// After:
public class MyService
{
    private const string ConnectionStringsDefault = "ConnectionStrings:Default";
    public void Process()
    {
        var a = ConnectionStringsDefault;
    }
}
```

### Replace with known constant

Available when a `DSA032_StringReplacements.txt` additional file defines a mapping for the string value. Replaces all matching occurrences in scope with the mapped expression.

```text
# DSA032_StringReplacements.txt
`ConnectionStrings:Default` -> AppConstants.DefaultConnectionString
```

```csharp
// Before:
var a = "ConnectionStrings:Default";

// After:
var a = AppConstants.DefaultConnectionString;
```

## Special cases

### Expression-bodied members

For expression-bodied methods and properties (e.g., `public string Name => "value";`), only the **class field** extraction is offered. Local constant extraction requires a block body.

```csharp
// Before:
public class MyService
{
    public string GetName() => "default name";
}

// After:
public class MyService
{
    private const string DefaultName = "default name";
    public string GetName() => DefaultName;
}
```

### Name generation

The constant name is generated from the string value using PascalCase:

| String value             | Generated name            |
|:-------------------------|:--------------------------|
| `"hello world"`          | `HelloWorld`              |
| `"connection-string:db"` | `ConnectionStringDb`      |
| `"123abc"`               | `StringConstant` (fallback for digit-leading names) |

If the generated name conflicts with an existing variable or field, a numeric suffix is appended (e.g., `HelloWorld1`).

### Nested classes

The class field is always inserted in the **innermost** containing type declaration, not an outer class.

## Not offered when

```csharp
// Empty string:
var a = "";

// Interpolated string:
var a = $"hello {name}";

// Non-string literal:
var a = 42;
```
