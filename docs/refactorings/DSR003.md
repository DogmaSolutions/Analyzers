# DSR003

Split file with multiple top-level types into one file per type.

| Property       | Value                                              |
|:---------------|:---------------------------------------------------|
| Refactoring ID | DSR003                                             |
| Title          | Split file into N files (one per type)             |
| Language       | C#                                                 |

## Description

This refactoring splits a source file containing multiple top-level type declarations (classes, structs, interfaces, enums, records) into separate files — one per type. Each new file is named after its type (e.g., `OrderService.cs`). All `using` directives are conservatively copied to each new file. Nested types remain inside their parent.

Unlike the [DSA033](../rules/DSA033.md) code fix (which only appears when the file exceeds a line count threshold), this refactoring is available from the IDE lightbulb menu on **any** file with two or more top-level types, regardless of file size.

## Trigger

The refactoring is offered when the cursor is on any top-level type declaration and the file contains **two or more** top-level type declarations.

## Actions offered

### Split file into N files (one per type)

Each top-level type is extracted into its own file. The original file is renamed to match its remaining type.

```csharp
// Before: Models.cs (3 types in one file)
using System;

namespace MyApp
{
    public class Customer { ... }
    public class Invoice { ... }
    public class Product { ... }
}

// After: Customer.cs
using System;

namespace MyApp
{
    public class Customer { ... }
}

// After: Invoice.cs
using System;

namespace MyApp
{
    public class Invoice { ... }
}

// After: Product.cs
using System;

namespace MyApp
{
    public class Product { ... }
}
```

Supports block namespaces, file-scoped namespaces, and files with no namespace.

## Not offered when

```cs
// File contains only one top-level type:
namespace MyApp
{
    public class OnlyOne { ... }
}

// File contains no type declarations:
using System;
// just using directives or top-level statements
```
