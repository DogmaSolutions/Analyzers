# DSR001

Insert `.TagWithCallSite()` or `.TagWith("...")` before any Entity Framework query terminal method.

| Property     | Value                                                                                          |
|:-------------|:-----------------------------------------------------------------------------------------------|
| Refactoring ID | DSR001                                                                                      |
| Title        | Insert `.TagWithCallSite()` or `.TagWith("...")` before terminal method                        |
| Language     | C#                                                                                             |

## Description

This refactoring inserts a call to `.TagWithCallSite()` or `.TagWith("...")` immediately before the terminal materialization method (such as `ToListAsync`, `FirstOrDefaultAsync`, `CountAsync`, `ToList`, etc.) on an Entity Framework Core LINQ query chain.

Unlike the [DSA021](../rules/DSA021.md) analyzer, this refactoring is available even when the DSA021 rule is suppressed, disabled, or set to a severity that does not trigger code fixes. It appears in the IDE lightbulb/screwdriver menu whenever the cursor is on an eligible terminal method.

## Trigger

The refactoring is offered when the cursor is on an EF Core terminal method that meets **all** of the following conditions:

1. The method name is a recognized terminal method (`ToListAsync`, `ToArrayAsync`, `FirstOrDefaultAsync`, `CountAsync`, `AnyAsync`, `ToList`, `FirstOrDefault`, `SingleOrDefault`, etc.)
2. The query chain originates from an Entity Framework `DbSet<T>` (directly, through a variable, or through LINQ operators)
3. The query chain does not already contain a call to `TagWith` or `TagWithCallSite`
4. The cursor is not inside a subquery lambda (e.g., a `Where` predicate containing a nested EF query)

## Actions offered

### Insert `.TagWithCallSite()` before terminal method

Inserts `.TagWithCallSite()` between the last query operator and the terminal method. `TagWithCallSite` automatically embeds the source file path, line number, and calling method name into the generated SQL as a comment.

```csharp
// Before:
await _context.Users.Where(u => u.IsActive).ToListAsync();

// After:
await _context.Users.Where(u => u.IsActive).TagWithCallSite().ToListAsync();
```

### Insert `.TagWith("...")` before terminal method

Inserts `.TagWith("TODO: describe this query")` with a placeholder string for the developer to replace with a meaningful label.

```csharp
// Before:
await _context.Users.Where(u => u.IsActive).ToListAsync();

// After:
await _context.Users.Where(u => u.IsActive).TagWith("TODO: describe this query").ToListAsync();
```

## Not offered when

```csharp
// Already tagged with TagWithCallSite:
_context.Users.TagWithCallSite().Where(u => u.IsActive).ToListAsync()

// Already tagged with TagWith:
_context.Users.Where(u => u.IsActive).TagWith("my query").ToListAsync()

// In-memory collection (not Entity Framework):
var list = new List<User>();
list.Where(u => u.IsActive).ToList()

// Cursor on a non-terminal method (e.g., Where, OrderBy):
_context.Users.Where(u => u.IsActive).ToListAsync()
//             ^ cursor here — no refactoring

// Inside a subquery lambda:
_context.Users
    .Where(u => _context.Orders.Any(o => o.UserId == u.Id))
    //                          ^ cursor here — no refactoring
    .TagWithCallSite()
    .ToListAsync();
```
