# DSR004

Split a single-type file into partial files by visibility or by topic.

| Property       | Value                                                         |
|:---------------|:--------------------------------------------------------------|
| Refactoring ID | DSR004                                                        |
| Title          | Split into partial files by member visibility / by topic      |
| Language       | C#                                                            |

## Description

This refactoring splits a source file containing a single top-level type into multiple partial-class files. Two strategies are offered: grouping members by access modifier (visibility) or by recurring words in member names (topic).

Unlike the [DSA034](../rules/DSA034.md) code fix (which only appears when the file exceeds a line count threshold), this refactoring is available from the IDE lightbulb menu on **any** single-type file with at least 2 members, regardless of file size.

## Trigger

The refactoring is offered when the cursor is on a type declaration and the file contains **exactly one** top-level type with **at least 2 members**.

## Actions offered

### Split into partial files by member visibility

Members are grouped by their effective access modifier:

- **`TypeName.Ctors.cs`** — constructors, destructors, `Dispose`/`DisposeAsync`, all fields, and all properties. Base types and interfaces are declared only on this file.
- **`TypeName.Public.cs`** — public methods
- **`TypeName.Internal.cs`** — internal methods
- **`TypeName.Protected.cs`** — protected methods
- **`TypeName.Private.cs`** — private methods
- **`TypeName.ProtectedInternal.cs`** — protected internal methods
- **`TypeName.PrivateProtected.cs`** — private protected methods
- **`TypeName.NestedTypeName.cs`** — each nested type gets its own file

Empty groups are omitted (no file is created for a visibility level with no members).

```csharp
// Before: OrderService.cs
namespace MyApp
{
    public class OrderService
    {
        private readonly IRepository _repo;
        public OrderService(IRepository repo) { _repo = repo; }
        public void ImportOrder() { ... }
        private void ValidateOrder() { ... }
    }
}

// After: OrderService.Ctors.cs (fields, properties, constructors)
// After: OrderService.Public.cs (public methods)
// After: OrderService.Private.cs (private methods)
```

### Split into partial files by topic

Members are grouped by recurring words extracted from their PascalCase names:

- **`TypeName.Ctors.cs`** — constructors, destructors, `Dispose`/`DisposeAsync` only (fields and properties are assigned by topic)
- **`TypeName.TopicName.cs`** — members whose names share the same topic word (e.g., `ImportOrder`, `ExportOrder` share the topic "Order")
- **`TypeName.Misc.cs`** — members that don't match any top topic
- **`TypeName.NestedTypeName.cs`** — each nested type gets its own file

Topic detection uses PascalCase word decomposition with a configurable list of excluded stop words (common verbs like Get, Set, Create, etc.). The top N topics by frequency are selected. Members matching multiple topics are assigned to the highest-frequency one. See [DSA034 configuration](../rules/DSA034.md#rule-configuration) for `max_topics` and `excluded_topic_words` settings.

```csharp
// Before: OrderService.cs
namespace MyApp
{
    public class OrderService
    {
        public void ImportOrder() { ... }
        public void ExportOrder() { ... }
        public void GrantPermission() { ... }
        public void RevokePermission() { ... }
        public void ClearCache() { ... }
    }
}

// After: OrderService.Order.cs (ImportOrder, ExportOrder)
// After: OrderService.Permission.cs (GrantPermission, RevokePermission)
// After: OrderService.Misc.cs (ClearCache — no top topic match)
```

## Not offered when

```cs
// File contains two or more top-level types (use DSR003 instead):
public class Alpha { }
public class Beta { }

// Single type with fewer than 2 members:
public class Tiny
{
    public int Value { get; set; }
}

// File contains no type declarations
```
